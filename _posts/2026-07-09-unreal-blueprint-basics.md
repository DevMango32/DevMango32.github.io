## 블루프린트란?

언리얼 엔진이 제공하는 **블루프린트(Blueprint)**는 비주얼 스크립팅 시스템으로, 복잡한 C++ 문법 없이 노드를 연결하는 방식으로 코딩할 수 있다.

실제 프로젝트에서는 C++ 기반으로 코딩하는 경우가 많지만, 블루프린트도 장점이 많다. 특히 **간단한 기능 구현이나 프로토타이핑**을 할 때 아주 유용하다.

![블루프린트 에디터](/assets/img/blueprint.png)

## 에디터 기본 조작

| 조작 | 기능 |
|---|---|
| 우클릭 유지 + WASD | 뷰포트 이동 |
| Q | 선택 모드 |
| W | 위치 이동 (화살표 중앙을 잡으면 두 축 동시 이동) |
| E | 회전 |
| R | 크기 조절 |
| End | 오브젝트를 자동으로 땅에 배치 |
| F | 선택한 오브젝트로 카메라 줌 |

- **Content Browser**: 프로젝트의 리소스(에셋) 관리
- **Place Actors**: 큐브, 라이트 등 기본 리소스 배치

## 주요 이벤트

| 이벤트 | 실행 시점 |
|---|---|
| BeginPlay | 게임 실행 시 처음 한 번만 실행 |
| Tick | 매 프레임마다 실행 (60프레임이면 1초에 60번) |

## 블루프린트 에디터 조작

| 조작 | 기능 |
|---|---|
| 우클릭 유지 | 그래프 공간 이동 |
| 우클릭 한 번 | 노드(이벤트/함수) 검색 |
| 드래그 선택 + C | 코멘트 달기 |
| 드래그 선택 + Q | 노드 일직선 정렬 |
| 선 더블클릭 | 중간점 생성 → 선 정리 가능 |
| Alt + 핀 클릭 | 선 연결 끊기 |
| Ctrl + 변수 드래그 | Get 노드 생성 |
| Alt + 변수 드래그 | Set 노드 생성 |

## 자주 쓰는 노드

- **Print Text / Print String**: 화면에 텍스트 출력 (print문과 동일)
- **Branch**: if문과 같다. 자주 쓰므로 `B + 좌클릭`으로 바로 생성
- **Sequence**: `S + 좌클릭`으로 생성. break 없는 switch와 비슷하게 0번부터 지정한 번호까지 순서대로 실행. 노드 정리 차원에서 사용
- **FlipFlop**: 호출할 때마다 A와 B를 번갈아 실행
- **Format Text**: 텍스트 조합
- **Get / Set**: 변수 읽기/쓰기
- 사칙연산, 비교 연산자, AND / OR / NOT
- Keyboard Events / Mouse Events
  - **Pressed**: 버튼을 누를 때
  - **Released**: 버튼을 뗄 때

### 흐름 제어 노드

| 노드 | 기능 |
|---|---|
| Gate | 열림/닫힘 상태에 따라 실행 통과 여부 결정 |
| MultiGate | 호출할 때마다 여러 출력 핀을 순서대로 실행 |
| Do Once | 딱 한 번만 실행 |
| Do N | N번까지만 실행 |

## 디버깅

- 노드 선택 + **F9**: 브레이크포인트 설정
- **F10**: 다음 노드로 한 단계씩 실행
